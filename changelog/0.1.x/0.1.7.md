# 0.1.7 — 2026-06-12

## Added

- **Truncation enrichment** on `smithsonian_search`, `smithsonian_explore`, `smithsonian_list_terms`, and `smithsonian_find_related` — each now declares an `enrichment` block (`truncated`, `shown`, `cap`, and where applicable `truncationCeiling`) and emits `ctx.enrich.truncated()` when the returned set is smaller than the total available, so a client can tell a capped page from a complete one.
- **`scripts/clean-mcpb.ts`** — post-pack MCPB cleaner wired into `bun run bundle`; runs `mcpb clean` (dev-dependency prune) then strips dependency-shipped agent docs (`node_modules/**` `skills/`, `.claude/`, `.agents/`, `SKILL.md`) that root-anchored `.mcpbignore` patterns cannot reach.
- **Docker `HEALTHCHECK`** probing `/healthz`, plus an `org.opencontainers.image.version` label fed by the `APP_VERSION` build arg.

## Changed

- **Error codes** on the `invalid_id` / `invalid_filter` contracts across `smithsonian_search`, `smithsonian_get_object`, `smithsonian_get_media`, and `smithsonian_find_related` migrated from `JsonRpcErrorCode.InvalidParams` to `JsonRpcErrorCode.ValidationError`.
- **Server identity** — `createApp()` now sets `name` and `title` explicitly to `smithsonian-mcp-server` rather than relying on package-name fallback (which would surface the npm scope).
- **`scripts/check-framework-antipatterns.ts`** adds a `z.coerce.boolean()` env-flag rule (use `z.stringbool()`) and skips comment lines so JSDoc mentions of an antipattern no longer register as violations.
- **`scripts/lint-packaging.ts`** extended with bundle-content guards (unanchored `.mcpbignore` runtime-path bypass, post-bundle `node_modules` agent-doc check) and a `createApp()`/manifest `display_name` identity check.
- **`.mcpbignore`** root dev-directory patterns anchored with a leading `/` so they exclude only the bundle root, not matching paths nested under `node_modules/`.
- **Vendored skills** re-synced to the installed framework version; `CLAUDE.md` / `AGENTS.md` drop the removed `ctx.sample` references; new `skills/techniques` reference added.

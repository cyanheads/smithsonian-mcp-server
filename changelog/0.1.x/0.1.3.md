# 0.1.3 — 2026-05-31

## Fixed

- **Structured filters on `smithsonian_search` and `smithsonian_explore` were silently ignored.** EDAN has no `fq` parameter — unknown query params are dropped. Field constraints are now embedded in `q` as ANDed Lucene `field:value` terms so they work as hard constraints. Multi-word values are quoted (`culture:"Plains Indian"`). ([#1](https://github.com/cyanheads/smithsonian-mcp-server/issues/1))
- **`smithsonian_explore` museum mode returned the entire catalog** instead of the requested unit's records. Museum mode now passes only `unit_code:X` as the filter with an empty query; the `query="*"` fallback that caused the full-catalog bleed is removed. ([#1](https://github.com/cyanheads/smithsonian-mcp-server/issues/1))
- **`smithsonian_find_related` returned hits from only the first fan-out signal.** The concat-then-slice merge filled the entire result limit from the first signal before others contributed. Results now round-robin interleave across signals so every fan-out contributes before any one backfills. ([#2](https://github.com/cyanheads/smithsonian-mcp-server/issues/2))

## Removed

- **`online_media_type` filter** removed from `smithsonian_search`. Live API testing confirms EDAN's search index does not support per-media-type filtering — all enum values (`Images`, `Videos`, `Audio`, `3D Images`) return zero results. The `online_only` boolean (`online_media_type:*` wildcard) and `cc0_only` boolean (`media_usage:CC0`) are unaffected and work correctly.

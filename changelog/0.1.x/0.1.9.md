# 0.1.9 — 2026-06-20

Framework-maintenance release: adopts `@cyanheads/mcp-ts-core` `^0.10.6 → ^0.10.9` and re-syncs the vendored tooling and skills. No server source or tool-surface changes.

## Added

- **`check-dependency-specifiers` devcheck step** (`scripts/check-dependency-specifiers.ts`, registered as `Dependency Specifiers`) — hard-fails on floating specifiers (`latest`, `*`, pre-release dist-tags) in `package.json`'s dependency sections and `bun.lock`'s `workspaces` map. Static and local, runs in the default and `--fast` passes. (cyanheads/mcp-ts-core#246)
- **Plugin marketplace manifest checks in `lint:packaging`** — validates `.claude-plugin/plugin.json` and `.codex-plugin/plugin.json` (non-empty descriptions, unscoped machine name in display fields, full scoped name in the install arg); gated by the new `devcheck.config.json` `packaging.pluginManifests` flag. The `devcheck` "Packaging" gate now also runs when a plugin manifest is present. (cyanheads/mcp-ts-core#240)
- **`ctx.content`** collector available from the framework — pushes image/audio media blocks into `content[]` only, never `structuredContent`; no-op until called, so existing tool output is byte-identical. (cyanheads/mcp-ts-core#239)

## Changed

- **`devcheck.config.json`** — added `packaging.pluginManifests: true`.
- **Fresh-scaffold devcheck guards** re-synced — changelog-sync `--check` exits cleanly on an empty `changelog/` tree; TODOs/FIXMEs, Tracked Secrets, and Framework Antipatterns skip via a shared `isGitRepo()` guard when `.git` is absent; `check-skill-versions.ts` skips a `SKILL.md` deleted from the worktree. (cyanheads/mcp-ts-core#242, cyanheads/mcp-ts-core#243, cyanheads/mcp-ts-core#237)
- **Canvas SQL gate** now classifies a `SELECT`-shaped statement that fails to prepare for a non-missing-table reason as `ValidationError` with `data.reason: 'invalid_sql'`, preserving the DuckDB binder detail. (cyanheads/mcp-ts-core#236)
- **`DuckdbProvider.describe({ tableName })`** — qualified the pushed `WHERE` filters with the table alias, fixing an ambiguous-column Binder Error on filtered `describe()`. (cyanheads/mcp-ts-core#235)
- **Vendored `skills/` re-synced** to the installed framework version; seven framework skill bodies advanced their `metadata.version`. (cyanheads/mcp-ts-core#238)

## Dependencies

- `@cyanheads/mcp-ts-core` `^0.10.6 → ^0.10.9`
- `@types/node` `^25.9.3 → ^26.0.0`

# 0.1.0 — 2026-05-30

Initial release of smithsonian-mcp-server, wrapping the [Smithsonian Open Access API](https://edan.si.edu/openaccess/apidocs/).

**API key required:** a free key from [api.data.gov](https://api.data.gov/signup) must be set as `SMITHSONIAN_API_KEY` in the environment. The key is sent via the `X-Api-Key` request header — never embedded in URLs.

## Added

- **`smithsonian_search`** — full-text search across 19.4 million objects from 20+ Smithsonian museums. Accepts structured filters: unit code, object type, decade, culture, geographic place, media type, online-only, and CC0-only. Returns title, museum, thumbnail URL, CC0 flag, and `record_id` for downstream tools. DataCanvas spillover activates when `rows > 20` or `canvas_id` is supplied.
- **`smithsonian_get_object`** — full catalog record by `record_id`: dates, makers, materials, dimensions, culture, topics, place, exhibitions, credit line, accession identifiers, rights statement, and a media summary with CC0 flag.
- **`smithsonian_get_media`** — CC0 images for a Smithsonian object at all available resolutions (thumbnail ~120px, screen ~800px, high-res JPEG/TIFF). Gated strictly to CC0 content — throws `Forbidden` when the object has media but none is CC0.
- **`smithsonian_explore`** — guided category browse answering "what does the Smithsonian have about X?" Four modes: `museum` (by unit code or full name), `culture`, `period` (decade), `medium` (object type). Returns total count, sample objects, and a museum breakdown.
- **`smithsonian_find_related`** — cross-museum discovery starting from an anchor `record_id`. Fans out up to 4 parallel searches using culture, maker, topic, and period+type signals extracted from the anchor; deduplicates and ranks results by number of matching signals.

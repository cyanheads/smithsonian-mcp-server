# 0.1.2 — 2026-05-31

## Removed

- **DataCanvas / DuckDB spillover** on `smithsonian_search` — the canvas path spilled to `canvas_id` + `table_name` outputs that had no consuming `dataframe_query` tool and always threw when `CANVAS_PROVIDER_TYPE=duckdb` was set without `@duckdb/node-api` installed. Drops the `canvas_id` input and `canvas_id` / `table_name` outputs. Resolves [#5](https://github.com/cyanheads/smithsonian-mcp-server/issues/5).
- **`services/canvas-accessor.ts`** — DataCanvas accessor module removed; `setCanvas` wiring removed from `index.ts`.

## Fixed

- **`smithsonian_search` with `rows > 20`** no longer throws on the hosted instance when `CANVAS_PROVIDER_TYPE=duckdb` is set. Returns up to `rows` (≤100) object summaries directly. Resolves [#3](https://github.com/cyanheads/smithsonian-mcp-server/issues/3) (superseded by [#5](https://github.com/cyanheads/smithsonian-mcp-server/issues/5)).

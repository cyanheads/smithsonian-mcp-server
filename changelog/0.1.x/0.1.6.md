# 0.1.6 — 2026-06-04

## Added

- **`smithsonian_list_terms`** tool — enumerate the valid term vocabulary for an indexed Smithsonian filter field before passing filter values to `smithsonian_search` or `smithsonian_explore`. Supported fields: `unit_code`, `object_type`, `culture`, `place`, `date`, `media_usage`, `online_media_type`. Returns terms sorted by object count descending; paginate with `start` + `rows` (max 100). Resolves [#6](https://github.com/cyanheads/smithsonian-mcp-server/issues/6).
- **`SmithsonianService.listTerms()`** — calls `/terms/{field}` and normalizes the response into `{ terms, total }`.
- **`RawTermsResponse`** type in `src/services/smithsonian/types.ts` — shapes the `/terms` endpoint response.

## Changed

- **`smithsonian_search` filter descriptions** updated to direct users to `smithsonian_list_terms` for vocabulary discovery (replacing "Search first to discover valid values").
- **`invalid_filter` error recovery hint** in `smithsonian_search` now explicitly names `smithsonian_list_terms` as the resolution step.
- **Server instructions** extended with a `smithsonian_list_terms` usage note.

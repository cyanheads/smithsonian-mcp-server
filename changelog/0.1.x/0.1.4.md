# 0.1.4 — 2026-05-31

## Added

- **`media_summary.cc0_image_count`** on `smithsonian_get_object` output — the number of CC0 images `smithsonian_get_media` returns for the same object. Computed through the same image pipeline (`extractImageItems` → CC0 filter), so the two counts reconcile by construction. ([#4](https://github.com/cyanheads/smithsonian-mcp-server/issues/4))

## Changed

- **`media_summary.has_cc0_images`** now derives from `cc0_image_count` (CC0 image-type items only) rather than the broader CC0 media count. Reads `false` when an object's only CC0 media are non-images (e.g. 3D models), matching what `smithsonian_get_media` would return. ([#4](https://github.com/cyanheads/smithsonian-mcp-server/issues/4))

## Fixed

- **`media_summary.count` vs `smithsonian_get_media` count discrepancy** — `count` is the raw `online_media.mediaCount` (all types), while `get_media` returns only CC0 images. For `nasm_A19700102000`, this produced a silent 16 vs 14 gap. The new `cc0_image_count` field makes the denominator difference legible. ([#4](https://github.com/cyanheads/smithsonian-mcp-server/issues/4))
